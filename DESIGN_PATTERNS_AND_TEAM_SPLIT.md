# Design Patterns and 3-Person Project Breakdown

## Scope
This document covers the Spring Boot runtime source set in src/main/java and the Thymeleaf templates in src/main/resources/templates. The previous duplicate non-Maven source mirror under src/parkinglot has been retired to keep a single authoritative code path.

## Design Patterns Used (with file locations and details)
- **Singleton** - `ParkingLot` uses a private constructor and double-checked locking in `getInstance` to ensure a single shared lot instance across the app. Files: [src/main/java/parkinglot/core/ParkingLot.java](src/main/java/parkinglot/core/ParkingLot.java); used by [src/main/java/com/team12/parkinglot_web/service/ParkingApplicationService.java](src/main/java/com/team12/parkinglot_web/service/ParkingApplicationService.java) and [src/main/java/com/team12/parkinglot_web/service/ParkingLotBootstrapService.java](src/main/java/com/team12/parkinglot_web/service/ParkingLotBootstrapService.java).
- **Factory** - `SpotFactory.createSpot` centralizes creation of `ParkingSpot` by ID and `SpotType`, keeping construction logic in one place. Files: [src/main/java/parkinglot/spots/SpotFactory.java](src/main/java/parkinglot/spots/SpotFactory.java); used by [src/main/java/com/team12/parkinglot_web/service/ParkingLotBootstrapService.java](src/main/java/com/team12/parkinglot_web/service/ParkingLotBootstrapService.java) and [src/main/java/parkinglot/Main.java](src/main/java/parkinglot/Main.java).
- **Observer** - `DisplayBoard` implements `Observer`, and `ParkingFloor.updateDisplay` counts free spots and notifies the display board on spot changes. Files: [src/main/java/parkinglot/interfaces/Observer.java](src/main/java/parkinglot/interfaces/Observer.java), [src/main/java/parkinglot/core/DisplayBoard.java](src/main/java/parkinglot/core/DisplayBoard.java), [src/main/java/parkinglot/core/ParkingFloor.java](src/main/java/parkinglot/core/ParkingFloor.java).
- **Strategy** - `Payment` delegates `processPayment` to a runtime-selected `PaymentStrategy` such as `CreditCardPayment`, `CashPayment`, or `NetBankingPayment`. Files: [src/main/java/parkinglot/interfaces/PaymentStrategy.java](src/main/java/parkinglot/interfaces/PaymentStrategy.java), [src/main/java/parkinglot/transactions/Payment.java](src/main/java/parkinglot/transactions/Payment.java), [src/main/java/parkinglot/transactions/CreditCardPayment.java](src/main/java/parkinglot/transactions/CreditCardPayment.java), [src/main/java/parkinglot/transactions/CashPayment.java](src/main/java/parkinglot/transactions/CashPayment.java), [src/main/java/parkinglot/transactions/NetBankingPayment.java](src/main/java/parkinglot/transactions/NetBankingPayment.java).
- **Adapter** - `PaymentProcessorAdapter` translates web-layer `PaymentRequest` values into a domain `Payment` with an appropriate `PaymentStrategy` and validates the form fields before execution. Files: [src/main/java/com/team12/parkinglot_web/payment/PaymentProcessorAdapter.java](src/main/java/com/team12/parkinglot_web/payment/PaymentProcessorAdapter.java), [src/main/java/com/team12/parkinglot_web/payment/PaymentRequest.java](src/main/java/com/team12/parkinglot_web/payment/PaymentRequest.java), [src/main/java/parkinglot/transactions/Payment.java](src/main/java/parkinglot/transactions/Payment.java).

## 3-Part Ownership Map (who explains what and where it lives)

### Person 1 - Web UI and Controller Flow
- Scope: routing, session guards, and view rendering for all user-facing screens.
- Feature walkthrough: registration and login flows; session-based dashboard access; live spot map page; single-spot booking action; multi-spot checkout; payment form handoff; success page; booking history page rendered from ticket data.
- Implementation notes: `ParkingController` gates protected routes with a session `userEmail`, builds page models, and constructs a `PaymentRequest` from form inputs for downstream processing.
- Code files: [src/main/java/com/team12/parkinglot_web/ParkingController.java](src/main/java/com/team12/parkinglot_web/ParkingController.java), [src/main/java/com/team12/parkinglot_web/ParkinglotWebApplication.java](src/main/java/com/team12/parkinglot_web/ParkinglotWebApplication.java), [src/main/java/com/team12/parkinglot_web/payment/PaymentRequest.java](src/main/java/com/team12/parkinglot_web/payment/PaymentRequest.java), [src/main/resources/templates/index.html](src/main/resources/templates/index.html), [src/main/resources/templates/login.html](src/main/resources/templates/login.html), [src/main/resources/templates/register.html](src/main/resources/templates/register.html), [src/main/resources/templates/dashboard.html](src/main/resources/templates/dashboard.html), [src/main/resources/templates/spots.html](src/main/resources/templates/spots.html), [src/main/resources/templates/checkout.html](src/main/resources/templates/checkout.html), [src/main/resources/templates/payment.html](src/main/resources/templates/payment.html), [src/main/resources/templates/success.html](src/main/resources/templates/success.html), [src/main/resources/templates/bookings.html](src/main/resources/templates/bookings.html).

### Person 2 - Application Services, Payment Orchestration, and Persistence
- Scope: business validation, pricing rules, booking concurrency control, payment orchestration, vehicle-session persistence, and startup reconciliation.
- Feature walkthrough: checkout summary calculation uses $totalFee = spotCount * hours * HOURLY_RATE$; `bookingLock` prevents race conditions during booking/payment; tickets persist vehicle and entry/exit lifecycle fields; BCrypt is used for secure password storage; startup sync rehydrates active bookings from SQLite.
- Implementation notes: `ParkingApplicationService` validates inputs, resolves spot availability, and updates displays; `DatabaseHelper` manages schema/migrations, lifecycle status updates, and active-session lookup; `ParkingLotBootstrapService` creates floors/spots and restores active reservations.
- Code files: [src/main/java/com/team12/parkinglot_web/service/ParkingApplicationService.java](src/main/java/com/team12/parkinglot_web/service/ParkingApplicationService.java), [src/main/java/com/team12/parkinglot_web/service/ParkingLotBootstrapService.java](src/main/java/com/team12/parkinglot_web/service/ParkingLotBootstrapService.java), [src/main/java/com/team12/parkinglot_web/payment/PaymentProcessorAdapter.java](src/main/java/com/team12/parkinglot_web/payment/PaymentProcessorAdapter.java), [src/main/java/parkinglot/db/DatabaseHelper.java](src/main/java/parkinglot/db/DatabaseHelper.java), [src/test/java/com/team12/parkinglot_web/ParkingControllerTest.java](src/test/java/com/team12/parkinglot_web/ParkingControllerTest.java), [src/test/java/com/team12/parkinglot_web/service/ParkingApplicationServiceTest.java](src/test/java/com/team12/parkinglot_web/service/ParkingApplicationServiceTest.java), [src/test/java/com/team12/parkinglot_web/ParkinglotWebApplicationTests.java](src/test/java/com/team12/parkinglot_web/ParkinglotWebApplicationTests.java).
